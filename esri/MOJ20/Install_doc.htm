System Requirements

The system requirements and supported environments for MapObjects-Java are listed on the ESRI Support Center at http://support.esri.com. For direct access to the MapObjects-Java page, go here: http://support.esri.com/index.cfm?fa=software.filteredGateway&PID=22.


Java Virtual Machine conflict for ArcExplorer—Java, ArcIMS users

The sample applications that are included with MapObjects require a Java Virtual Machine (VM). The sample applications are included as part of the Typical MapObjects installation and are also available as an option during a Custom MapObjects installation. During both of these installation scenarios, you will be asked to install a Java VM specifically for use with these sample applications or to navigate to one already available on your system. If you are installing on a system that also runs ArcIMS and/or ArcExplorer—Java Edition, it is recommended that you choose to install the Java VM specifically for use with the MapObjects sample applications and compile script. If you navigate to and use the same Java VM used by ArcIMS and/or ArcExplorer, all three environments may destabilize, causing unexpected runtime results. This is because the ArcSDE JARs, JAI JARs and DLLs, JSSE JARs, and possibly other JAR files could potentially be mixed into the same JRE extension location as older versions of the same products. Even though the JAR file names may be different, the VM could possibly encounter the same package names within these JAR files. 


Mac OS X installation

To install on the Macintosh, navigate to the MOJava.app file in the Platforms\MacOSX folder. You must first copy this MOJava.app file to a local directory and then double-click on it to begin the InstallAnywhere process. (Double-clicking directly from the CD will not allow installation to begin.)


RedHat Linux 7.x installation issue

The MapObjects-Java install may fail due to the presence of an optional RedHat Linux package named Kaffe. This package is not installed by default, but you will have it if you request the RedHat Linux install program to install all packages.

When running the installer, you can tell if Kaffe is causing an issue if you see error messages like these:

Error: Unrecognized JVM specific option `-Xmx50331648'

and

java.compiler == kaffe.jit

To work around this issue, uninstall Kaffe and two other related packages—gcc3-java and gcc-java.

Installing the TrueType font for North Arrows

The TrueType font called esri_40.ttf is used by the Layout package for selecting North Arrow styles. This file can be found in the <MOJHOME>\MOJ20\Fonts folder. The other ESRI TrueType fonts can also be found in this folder as well.

The instructions for installing this font are as follows:

Windows

Copy the esri_40.ttf file to C:\WINNT\Fonts.

 

RedHat Linux

1. Copy the esri_40.ttf file to any directory location.

2. Construct a fonts.dir file using system utility ttmkfdir.

3. Add the directory to the X font server using system utility chkfontpath.

4. Restart X font server and restart X (reboot is easiest).

 

Sun Solaris

Copy the esri_40.ttf file to /usr/openwin/lib/X11/fonts/TrueType/.

 

IBM-AIX

Copy the esri_40.ttf file to /usr/lpp/X11/lib/X11/fonts/TrueType/.

 

Mac OS X

Copy the esri_40.ttf file to /Library/Fonts.

 

HP-UX

Copy the esri_40.ttf file to /usr/lib/X11/fonts/ms.st/typefaces.

Uninstallation information

Some files are not removed when you perform an uninstall at this release. You should manually delete these files after performing an uninstall of MapObjects.

Windows 2000 and XP
<Drive:>\Documents and Settings\<User>\Local Settings\Temp\*.tmp\remove.exe

Linux and Solaris
$HOME\IA_Installers
\tmp\remove.sh

Solaris
<Install_location>/jre/lib/sparc/client/libjsig.so
<Install_location>/jre/lib/sparc/server/libjsig.so

IBM-AIX
/tmp/install.dir.*

HP-UX
/tmp/ia_remove.sh
