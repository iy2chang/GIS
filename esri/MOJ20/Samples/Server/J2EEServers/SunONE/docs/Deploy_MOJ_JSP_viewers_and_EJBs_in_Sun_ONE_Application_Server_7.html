Deploy MOJ EJBs and JSP Viewer in Sun ONE Application Server

Start up Sun ONE application server

  1. Create a war file, output.war, that packs all of the contents under the folder $MOJ20/Samples/Server/J2EEServers/SunONE/output.

     prompt> jar -cf output.war *

    Note that the $MOJ20 denotes the MapObjects Java Edition 2.0’s installation directory such as D:\ESRI\MOJ20 on a Windows machine.
     
  2. Copy all of the jar files (and libraries) from $MOJ20/lib and $MOJ20/Samples/Server/IOStreamProvider/esri_iostreamprovider.jar to $SAS/domains/<domain_name>/<server_name>/lib (where $SAS denotes SunONE application server install location such as D:\Sun\AppServer7 in Windows machine). <domain_name> and <server_name> are the domain and server created by SunONE application server. For example, D:\Sun\AppServer7\domains\domain1\server1 is usually the first default server created in a Windows machine.  Note that the user account that installed SunONE application server or a root account is required to copy files from MOJ20 installation folder to SunONE application server's lib.  

  3. Start the SunONE application server if the server is already runing, you need to re-start the server again.

    For a Windows machine, from the startup menu, Start-> Programs->Sun Microsystems->Sun ONE Application Server 7->Start Application Server.

    For a Linux or Unix machine, go to $SAS/bin directory, run the following script,
    prompt> ./asadmin start-appserv  

  4. Start the SunONE administrative console:

    For a Windows machine,  from the startup menu, Start-> Sun Microsystems->Sun ONE Application Server 7->Start Admin Console. You will be see a window popup.

    For a Linux or Unix machine, open a browser and enter an URL in form of http://machine_name:4848/. 

    Enter the User Name and password and then click on OK. You should see the following screen shot:



  5. The following 4 steps are required for Linux and Unix machines only. Click on server1 (or server instance(s) you want to deploy the sample) in the left pane, then click on JVM Settings tab in the right pane. 



  6. Click on JVM Options, then type in following line in JVM Option:
    -Djava.awt.headless=true  



  7. Click on Add and then Save button on the bottom.



  8. Go back to the General tab and click on Restart button to re-start the server.

Deploy a web application as temporary map image folder 

  1. Click on the App Server Instances -> server1->Applications->Web Apps node in the left pane of the administrative console.


     

  2. Click on Deploy… and then Browse… button, then then navigate the file selection window to  $MOJ20/Samples/Server/J2EEServers/SunONE/output folder and select the output.war file. Then click OK.
     


  3. Keep the default values for the entries and checkboxes and then click OK.
     


  4. Now the web application of output.war is deployed. Note that there is a Location column in the deployed output application. This output location will be used as a temporary map image.  



Deploy the JSP viewers

  1. Replace the $MOJ20 string within the two sample AXL files (usa.axl and world.axl in $MOJ20/Samples/Server/AXLFiles folder) with the MOJ 2.0 install directory, such as ‘/ESRI/MOJ20’

    <ARCXML>
    ...
    <WORKSPACES>
    <SHAPEWORKSPACE name="ws-0" directory="$MOJ20/Samples/Data/USA" />
    </WORKSPACES>
    ...
    </ARCXML>

  2. Modify the two map service configuration files, mapservices.xml, located at  $MOJ20/Samples/Server/J2EEServers/SunONE/JspViewer/WEB-INF/classes and $MOJ20/Samples/Server/J2EEServers/SunONE/JspServletViewer/WEB-INF/classes. Replace $MOJ20 strings in the axl_file element with the MOJ2.0 install directory, such as '/ESRI/MOJ20'  and the #output_directory_name# strings with the map image output directory as shown in step 4 of Deploy a web application as temporary image. It looks something like this: 

    <services>
    <name>usa1</name>
    ...
    <output_dir>    $SAS/domains/<domain_name>/<instance_name>/applications/j2ee-modules/output1</output_dir>
    <output_url></output_url>
    <axl_file>$MOJ20/Samples/Server/AXLFiles/usa.axl</axl_file>
    ...
    <name>world2</name>
    ...
    <output_dir>    $SAS/domains/<domain_name>/<instance_name>/applications/j2ee-modules/output1</output_dir>
    <output_url></output_url>
    <axl_file>$MOJ20/Samples/Server/AXLFiles/world.axl</axl_file>
    ...
    </services>

  3. Use the jar command to create a war file named JspViewer.war that packs all of the files under $MOJ20/Samples/Server/J2EEServers/SunONE/JspViewer.

    prompt> jar -cf JspViewer.war *

  4. Use the jar command to create a war file named JspServletViewer.war that packs all of the files under $MOJ20/Samples/Server/J2EEServers/SunONE/JspServletViewer.

    prompt> jar -cf JspServletViewer.war *

  5. Go through steps similar to steps 1 through 3 from Deploy a web application as temporary image to deploy JspViewer (JspViewer.war located at $MOJ20/Samples/Server/J2EEServers/SunONE/JspViewer) and JspServletViewer (JspServletViewer.war located at $MOJ20/Samples/Server/J2EEServers/SunONE/JspServletViewer).

  6. Now you can start testing these two JSP viewers by following instructions in Testing JSP Viewers.

Deploy SimpleEJB and its viewer

  1. You can skip this step if you already did Step 1 in Deploy JSP Viewers. Otherwise, replace the $MOJ20 string within the two sample AXL files (usa.axl and world.axl) with the MOJ 2.0 install directory, such as ‘/ESRI/MOJ20’ .

    <ARCXML>
    ...
    <WORKSPACES>
    <SHAPEWORKSPACE name="ws-0" directory="$MOJ20/Samples/Data/USA" />
    </WORKSPACES>
    ...
    </ARCXML>

  2. Modify the map service configuration file, mapservices.xml, located at $MOJ20/Samples/Server/J2EEServers/SunONE/SimpleEjb/META-INF directory. Replace $MOJ20 strings in the axl_file element with the MOJ2.0 install diretory such as '/ESRI/MOJ20'  and the #output_directory_name# strings with the map image output directory as shown in step 4 of Deploy a web application as temporary image.
     

    <services>
    <name>usa1</name>
    ...
    <output_dir>$SAS/domains/<domain_name>/<instance_name>/applications/j2ee-modules/output1</output_dir>
    <output_url></output_url>
    <axl_file>$MOJ20/Samples/Server/AXLFiles/usa.axl</axl_file>
    ...
    <name>world2</name>
    ...
    <output_dir>$SAS/domains/<domain_name>/<instance_name>/j2ee-modules/output1</output_dir>
    <output_url></output_url>
    <axl_file>$MOJ20/Samples/Server/AXLFiles/world.axl</axl_file>
    ...
    </services>
     
  3. Create a jar file called SimpleEjb.jar that packs all of the files under directory of $MOJ20/Samples/Server/J2EEServers/SunONE/SimpleEjb.
     
  4. Copy SimpleEjb.jar, which you created in the last step and SimpleEjbViewer.war from $MOJ20/Samples/Server/J2EEServers/SunONE to the SimpleEjbEar folder. Then create an ear file called SimpleEjb.ear that packs SimpleEjb.jar, SimpleEjbViewer.war and META-INF folder with the jar command.
     
  5. Now deploy the ear file you created with the SunONE application server’s browser-based Admin Console. Start the Admin Console and click on the node App Server Instances -> <Instance_name> ->Applications->Enterprise Apps in the left pane of the console. You will see the following screen:
     


  6. Click on Deploy...  and then Browse… , and then navigate the file selection window to  $MOJ20/Samples/Server/J2EEServers/SunONE/SimpleEjbEar folder and select the SimpleEjb.ear file you created in Step 3.
     


  7. Click on OK to upload the selected ear file to the application server.
     


  8. Click on OK to deploy the application. You should see the following screen shot:



  9. Now you can start testing SimpleEJB and its viewer by following the instructions in Testing SimpleEJB and its Viewers.

Deploy AdvancedEJB and its viewer

    Create a Database Table of mapservice

    For MySQL Database
  1. Download and Install MySQL database from http://www.mysql.com by following the instructions from the following URL http://www.mysql.com/documentation/mysql/bychapter/manual_Installing.html.

  2. Start MySQL and then Create a table under database test (test is one of the default databases created by MySQL) with the following SQL script, (use mysql client and go to the test database with command “use test” under the mysql prompt).
     
    CREATE TABLE mapservice (name varchar(255) not null, type varchar(50), config blob, output_dir varchar(255), output_url varchar(255), image_format varchar(5))

  3. Download the JDBC driver for MySQL from http://www.mysql.com/products/connector-j/index.html and unzip it. Then copy the jar file, mysql-connector-java-3.0.6-stable-bin.jar, to the $SAS/domains/<domain_name>/<server_name>/lib folder.
    For Oracle Database
  1. Create a MAPSERVICE table using Oracle’s Enterprise Database Manager or SQLPlus. Please note that the CONFIG column data type is CLOB.
    CREATE TABLE "user_name"."MAPSERVICE" ("NAME" VARCHAR2(255) NOT NULL,
        "TYPE" VARCHAR2(50) NOT NULL, "CONFIG" CLOB NOT NULL,
        "OUTPUT_DIR" VARCHAR2(255), "OUTPUT_URL" VARCHAR2(255),
        "IMAGE_FORMAT" VARCHAR2(5) NOT NULL, UNIQUE("NAME")) 
        TABLESPACE "table_space_name"
  2. Get the JDBC driver from the Oracle server installation (usually classes12.jar file from folder like $ORACLE/ora92/jdbc/lib) and place it in the $SAS/domains/<domain_name>/<server_name>/lib folder.
    For Microsoft SQL Database
  1. Create a MAPSERVICE table. Please note that the CONFIG column data type is TEXT:
    CREATE TABLE mapservice (name varchar(255) not null, type varchar(50), config text, output_dir varchar(255), output_url varchar(255), image_format varchar(5))
  2. Download the JDBC driver from Microsoft's web site at http://www.microsoft.com/sql (3 jar files, msbase.jar, mssqlserver.jar and msutil.jar, extracted from either the downloaded tar file or installation program) and place them in the $SAS/domains/<domain_name>/<server_name>/lib folder.

    Create JDBC connection pool and JDBC data source

  1. Create JDBC connection pool and data source. Click on the App Server Instances-><Server_instance>->JDBC->Connection Pools node on the left pane of the server admin console.



  2. Click on New…  and then type the connection pool name in the Name field as MySQLConnectionPool. Select a Database Vendor from the dropdown list:
        Select Other for MySQL or Microsoft SQL Server, or  
        Select Oracle 9.x or Oracle 8.1.x for Oracle Database server.
    Then click Next to go to next screen.



  3. Enter the data source class name here:
        com.mysql.jdbc.jdbc2.optional.MysqlDataSource for MySQL database,
        oracle.jdbc.pool.OracleDataSource for Oracle database,
        com.microsoft.jdbcx.sqlserver.SQLServerDataSource for MS SQL database



  4. For a MySQL database, check the following 3 properties and enter a value for each one of them (note that you need the values from your local installation)
        serverName: localhost
        portNumber: 3306
        databaseName: test

    For an Oracle database, check the following 3 properties and enter a value for each one of them (note that you need the values from your local installation and the URL is a new entry not found in the default list)
        URL: jdbc:oracle:thin:@localhost:1521:db_sid
        user: UserName
        password: Password

    For a Microsoft SQL database, check the following 5 properties and enter a value for each one of them (note that you need the values from your local installation)
        serverName: localhost
        portNumber: 1433
        user: UserName
        password: Password
        databaseName: MyDatabaseName



  5. Keep everything else as default and click on Finish.



  6. Click the JDBC Resources node in the left pane and then click New… in the right pane. This will start process of creating a JDBC data source used by server sample EJBs.



  7. Enter MySQLDS in the JNDI Name entry and select MySQLConnectionPool from the Pool Name drop down list. Then click OK.



  8. Click on App Server Instances -> <Instance_name> node and Apply Changes and then re-start the server.



    9. Deploy AdvancedEJB and its Viewer

  9. Undeploying SimpleEJB sample if you already deployed it to avoid potential conflicts between these two EJB samples. 

  10. Remove two packages, com.esri.svr.cmn and com.esri.svr.ejb, from $MOJ20/Samples/Server/J2EEServers/SunONE/AdvancedEjbViewer/WEB-INF/classes and create a war file, AdvancedEjbViewer.war that packs all of the files under $MOJ20/Samples/Server/J2EEServers/SunONE/AdvancedEjbViewer. Then move the war file to the AdvancedEjbEar folder.

  11. Copy the AdvancedEjb.jar file from $MOJ20/Samples/Server/J2EEServers/SunONE to the $MOJ20/Samples/Server/J2EEServers/SunONE/AdvancedEjbEar.

  12. Go to the AdvancedEjbEar directory and create an ear file, AdvancedEjb.ear, which packs all of files under the AdvancedEjbEar folder with a jar command.

  13. Go through steps 5 through 8 in "Deploy SimpleEJB and its viewer" again to deploy the AdvancedEjb application with the ear file created in last step.  

  14. Skip this step if you want to use the web browser-based admin tool. Otherwise go through steps 5 through 8 in "Deploy SimpleEJB and its viewer" again to deploy the CatalogEjb.jar module from $MOJ20/Samples/Server/J2EEServers/SunONE. In Step 5, instead of clicking on Enterprise Apps, you should click on EJB Modules. Once you successfully deployed the CatalogEJB module, you can start testing AdvancedEJB and its viewer by following instructions in Testing AdvancedEJB and its viewer with stand alone admin tool.

    15. Deploy AdvancedEJB Browser-based Admin Tool

  15. Undeploy the CatalogEjb.jar module if you deployed CatalogEjb.jar in last step.

  16. Download the com.oreilly.servlet from http://www.servlets.com/cos/index.html and unzip the file. Copy the cos.jar file to $MOJ20/Samples/Server/J2EEServers/SunONE/AdvanedEjbAdmin/WEB-INF/lib.

  17. Remove two packages, com.esri.svr.ejb and com.esri.svr.cmn, from $MOJ20/Samples/Server/J2EEServers/SunONE/AdvancedEjbAdmin/WEB-INF/classes. Create a war file, AdvancedEjbAdmin.war that packs all of the files under $MOJ20/Samples/Server/J2EEServers/SunONE/AdvancedEjbAdmin. Then move the war file to the AdvancedEjbAdminEar folder.

  18. Copy the CatalogEjb.jar file from $MOJ20/Samples/Server/J2EEServers/SunONE to  $MOJ20/Samples/Server/J2EEServers/SunONE/AdvancedEjbAdminEar.

  19. Go to the AdvancedEjbAdminEar directory and create an ear file, AdvancedEjbAdmin.ear, which packs all of the files under the AdvancedEjbAdminEar folder with a jar command.

  20. Go through steps 5 through 8 in "Deploy SimpleEJB and its viewer" again to deploy AdvancedEjbAdmin application with the ear file created in last step.  

  21. Now you can start testing AdvancedEJB and its viewer by following instructions in Testing AdvancedEJB and its viewer with web-based admin tool.

Testing JSP viewers, map EJBs and their viewers

Testing JSP Viewers 

  1. Type the following URL in a browser to test JspViewer, http://localhost:<port_number>/JspViewer/viewer.htm.

  2. Type the following URL in a browser to test JspServletViewer, http://localhost:<port_number>/JspServletViewer/viewer.htm.

Testing SimpleEJB and its viewer

  1. Type the following URL in a browser, http://localhost:<port_number>/SimpleEjbViewer/viewer.htm. You should see the following screen shot that display a dropdown list showing the map services.



  2. Select ‘usa1’ and click ‘Get Map Service’. You should see the usa map display in the browser.



Testing the AdvancedEJB and its viewer with the standalone admin tool

  1. Remove the com/esri/svr/ejb folder from $MOJ20/Samples/Server/J2EEServers/SunONE/AdvancedEjbAdminApp. Pack the contents into a jar file, AdvancedEjbAdminApp.jar, under $MOJ20/Samples/Server/J2EEServers/SunONE/AdvancedEjbAdminApp using a jar command in a command window.

  2. To run the standalone admin tool, open a command window, and go the directory $MOJ20/Samples/Server/J2EEServers/SunONE. Type the following command when AdvancedEjbAdmin application deployed:

    for Windows
    java –classpath AdvancedEjbAdminApp.jar;$SAS\lib\appserv-ext.jar;$SAS\domains\<domain_name>\<server_name>\applications\j2ee-apps\AdvancedEjb_1\AdvancedEjbClient.jar;$SAS\domains\<domain_name>\<serverName>\applications\j2ee-apps\AdvancedEjbAdmin_1\AdvancedEjbAdminClient.jar com.esri.svr.admin.AdvancedEjbAdmin

    for Linux and Unix
    java –classpath AdvancedEjbAdminApp.jar:$SAS/lib/appserv-ext.jar:$SAS/domains/<domain_name>/<server_name>/applications/j2ee-apps/AdvancedEjb_1/AdvancedEjbClient.jar:$SAS/domains/<domain_name>/<serverName>/applications/j2ee-apps/AdvancedEjbAdmin_1/AdvancedEjbAdminClient.jar com.esri.svr.admin.AdvancedEjbAdmin

    Or type the following command when CatalogEJB is deployed as an EJB module:

    for Windows 
    java –classpath AdvancedEjbAdminApp.jar;$SAS\lib\appserv-ext.jar;$SAS\domains\<domain_name>\<server_name>\applications\j2ee-apps\AdvancedEjb_1\AdvancedEjbClient.jar;$SAS\domains\<domain_name>\<serverName>\applications\j2ee-modules\CatalogEjb_1\CatalogEjbClient.jar com.esri.svr.admin.AdvancedEjbAdmin

    for Linux and Unix
    java –classpath AdvancedEjbAdminApp.jar:$SAS/lib/appserv-ext.jar:$SAS/domains/<domain_name>/<server_name>/applications/j2ee-apps/AdvancedEjb_1/AdvancedEjbClient.jar:$SAS/domains/<domain_name>/<serverName>/applications/j2ee-modules/CatalogEjb_1/CatalogEjbClient.jar com.esri.svr.admin.AdvancedEjbAdmin

    You should see a Java application running with the following screen shot:



  3. Click on “Add Services” in the left pane to add a new service. Fill out the entries as shown in the following screen shot. The OutputURL is only required if you are using jCIFS or NFS file system to output the map images. Click on Add to add this new service to the database. If the service info has been successfully added into the database, you should see a message saying "The service 'world' has been successfully added". 



  4. Click on "List Services" in the left pane to show all of the deployed services.



  5. Select a service name, such as world, then click on Show Service Details to show information about the selected service.

         

  6. To test a service, open a browser and type the URL shown in the bottom of last screen in form of http://machinename:portnumber/AdvancedEjbViewer/viewer.htm?ServiceName=servicename, such as http://localhost:8084/AdvancedEjbViewer/viewer.htm?ServiceName=world

Testing the AdvancedEJB and its viewer with the web-based admin tool

  1. Type the following URL in a browser to bring up the admin, http://localhost:<port_number>/mojadmin/index.html



  2. Click on ‘Add Service’ to add a new map service. You need to fill out entries of ServiceName, ServiceType (right now it should be ‘image’), Image Format (PNG, JPG etc), Output Directory or OutputURL (for things like jCIFS and/or NFS servers), and the service configuration ArcXML file location or the ArcXML doc. Click Add after you have fill out required entries. If the service is successfully added, you should see a line say “The new service ‘world’ has been successfully added”.
     


  3. Click on "List Services" to get a list of all available map services.

       
  4. Click on the service name of world to list the contents of the service information.
     


  5. Click on hyperlink at the bottom of last screen, “Show world Map”
