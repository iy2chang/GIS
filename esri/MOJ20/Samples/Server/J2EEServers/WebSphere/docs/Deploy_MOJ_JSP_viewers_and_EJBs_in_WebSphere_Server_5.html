Deploy MOJ EJBs and JSP Viewer in WebSphere

Start up WebSphere example server

  1. Install WebSphere 5 in your local machine and note that the $WAS denotes the WebSphere Application Server’s installation directory such as D:\WebSphere\AppServer in Windows machine and $MOJ20 denotes the MapObjects Java Edition 2.0’s installation directory such as D:\ESRI\MOJ20 in a Windows machine in the following steps.

  2. Copy jar files from $MOJ20/lib and $MOJ20/Samples/Server/IOStreamProvider/esri_iostreamprovider.jar to the $WAS/lib/ext folder.

  3. Start WebSphere:

    For a Windows machine,  from startup menu, Start-> Programs->IBM WebSphere -> Application Server v5.0->Start the Server.

    For a Linux or Unix machine,  go to $WAS/bin directory, run the following script,
    prompt> ./startServer.sh <server_name>

  4. Start WebSphere administrative console:

    For a Windows machine,  from startup menu, Start->IBM WebSphere->Application Server v5.0->Administrative Console.

    For a Linux or Unix machine, open a browser and enter an URL in form of http://machine_name:port_number/admin (the default port for admin is 9090).   



    Enter User ID and then click on OK button, you should see the following screen shot.



Deploy a web application as temporary map image folder

  1. Create a war file under directory of $MOJ20/Samples/Server/J2EEServers/WebSphere/output folder by issuing the following command in a command window:

    prompt> jar –cf output.war WEB-INF

  2. The basic steps to deploy a web application follows the pattern of deploy the archive, save the configuration, and then restart the application server. Here is start of step-by-step instructions.

  3. Click on the Applications->Install New Application in the left pane of the administrative console.



  4. Click on Browse… to select the output.war file from $MOJ20/Samples/Server/J2EEServers/WebSphere/output and type output in the Context Root: entry. Then click on Next.



  5. Select Generate Default Bindings and Override existing bindings, then click Next.



  6. Keep all the default setting and click on the Next button to go step 2.



  7. Select Web Module and click on the Next button to go step 3.



  8. Select an application and click on the Module check box and then click on the Next button to go step 4.



  9. The summary information is listed in the final step as shown in the following screen shot. Click on the Finish button to complete the deployment of the output.war module.



  10. If you successfully deployed output.war module, you should the following screen shot. Click on the Save to Master Configuration link to save the configuration.



  11. Click on Enterprise Applications node in the left pane to see the deployed output.war module. Click on the check box next output_war and then click on Start button to start up the application. The red-cross sign should turn to green arrow sign. And you should the server configuration.



Deploy JSP viewers

  1. Replace the $MOJ20 string within the two sample AXL files (usa.axl and world.axl in $MOJ20/Samples/Server/AXLFiles folder) with the MOJ 2.0 install directory such as ‘/ESRI/MOJ20’.

    <ARCXML>
    ...
    <WORKSPACES>
    <SHAPEWORKSPACE name="ws-0" directory="$MOJ20/Samples/Data/USA" />
    </WORKSPACES>
    ...
    </ARCXML>

  2. Modifiy two map service configuration file, mapservices.xml, located at  $MOJ20/Samples/Server/J2EEServers/WebSphere/JspViewer/WEB-INF/classes directory and $MOJ20/Samples/Server/J2EEServers/WebSphere/JspServletViewer/WEB-INF/classes, replace $MOJ20 strings in the axl_file element with the MOJ2.0 install diretory such as '/ESRI/MOJ20'  and the #output_directory_name# strings with the map image output directory created in of Deploy a web application as temporary map image folder. It looks something like this: ( where $LOCAL_MACHINE_NAME denotes WebSphere install machine name).

    <services>
    <name>usa1</name>
    ...
    <output_dir>$WAS/AppServer/installedApps/$LOCAL_MACINE_NAME/output_war.ear/output.war    </output_dir>
    <output_url></output_url>
    <axl_file>$MOJ20/Samples/Server/AXLFiles/usa.axl</axl_file>
    ...
    <name>world2</name>
    ...
    <output_dir>$WAS/AppServer/installedApps/$LOCAL_MACINE_NAME/output_war.ear/output.war    </output_dir>
    <output_url></output_url>
    <axl_file>$MOJ20/Samples/Server/AXLFiles/world.axl</axl_file>
    ...
    </services>

  3. Use jar command to create a war file named JspViewer.war that packs all the files under directory of $MOJ20/Samples/Server/J2EEServers/WebSphere/JspViewer

  4. Use jar command to create a war file named JspServletViewer.war that packs all the files under directory of $MOJ20/Samples/Server/J2EEServers/WebSphere/JspServletViewer

  5. Go through the similar steps 3 through 11 as shown in the section of  Deploy a web application as a temporary map image folder to deploy JspViewer (JspViewer.war located at $MOJ20/Samples/Server/J2EEServers/WebSphere/JspViewer) and JspServletViewer (JspServletViewer.war locate at $MOJ20/Samples/Server/J2EEServers/WebSphere/JspServletViewer)

  6. Now you can start testing these two JSP viewers by following instructions in Testing JSP Viewers

Deploy SimpleEJB and its viewer

  1. You can skip this step if you already did the Step 1 in Deploy JSP Viewers, otherwise replace the $MOJ string within the two sample AXL files (usa.axl and world.axl) with the MOJ 2.0 install directory such as ‘/ESRI/MOJ20’

    <ARCXML>
    ...
    <WORKSPACES>
    <SHAPEWORKSPACE name="ws-0" directory="$MOJ/Samples/Data/USA" />
    </WORKSPACES>
    ...
    </ARCXML>

  2. Modify the map service configuration file, mapservices.xml located at $MOJ20/Samples/Server/J2EEServers/WebSphere/SimpleEjb/META-INF directory, replace $MOJ20 strings in the axl_file element with the MOJ2.0 install diretory such as '/ESRI/MOJ20'  and the #output_directory_name# strings with the map image output directory created in of Deploy a web application as temporary map image folder. It looks something like this ( Note that the $LOCAL_MACINE_NAME denotes the your local machine name where you installed the WebSphere application server).  

    <services>
    <name>usa1</name>
    ...
    <output_dir>$WAS/AppServer/installedApps/$LOCAL_MACINE_NAME/output_war.ear/output.war</output_dir>
    <output_url></output_url>
    <axl_file>$MOJ20/Samples/Server/AXLFiles/usa.axl</axl_file>
    ...
    <name>world2</name>
    ...
    <output_dir>$WAS/AppServer/installedApps/$LOCAL_MACINE_NAME/output_war.ear/output.war</output_dir>
    <output_url></output_url>
    <axl_file>$MOJ20/Samples/Server/AXLFiles/world.axl</axl_file>
    ...
    </services>


  3. Create a jar file called SimpleEjb.jar that packs all the contents under directory of $MOJ20/Samples/Server/J2EEServers/WebSphere/SimpleEjb.

  4. Now start to assemble a J2EE application that contains both the SimpleEjb and the SimpleEjbViewer with WebSphere’s Application Assembly Tool (ATT). Start the ATT by clicking on Start-> Programs->IBM WebSphere -> Application Server v5.0->Application Assembly Tool, you will see the following screen, change the display name to SimpleEjbAndViewer.



  5. Right Click on the EJB Modules and select Import menu item in the popup menu. Then navigate the file selection window to the $MOJ20/Samples/Server/J2EEServers/WebSphere/SimpleEjb folder and select the SimpleEjb.jar file you created in Step 3.



  6. Click on OK in the popup confirmation window (and optionally you can enter an alternate description).



  7. Right Click on the Web Modules and select Import menu item in the popup menu. Then navigate the file selection window to the $MOJ20/Samples/Server/J2EEServers/WebSphere folder and select the SimpleEjbViewer.war file. Type SimpleEjbViewer in the Context Root: field and then click OK.



  8. Save the application as SimpleEjbAndViewer.ear file in the folder of $MOJ20/Samples/Server/J2EEServers/WebSphere.



  9. Now go to WebSphere Administrative Console to deploy the SimpleEjbAndViewer.ear file. Click on the node of Applications->Install New Application and then click on the Browse… button and navigate to the folder of $MOJ20/Samples/Server/J2EEServers/WebSphere and select SimpleEjbAndViewer.ear file.
     


  10. Click on Next button to go next screen in which you check the top checkbox of  “Generate Default Bindings”
     


  11. Click Next button to go to Step1 where you check the Deploy EJBs checkbox and change Application Name entry to SimpleEjbAndViewer while keeping others as default values.



  12. Click on Next button and click Next button in the Step 2 screen without changing any default values.

    In the Step 3 screen, make sure EJB MapServiceI’s JNDI name is ejb/MapServiceI.



  13. Click Next button to go to Step 4 and check the Web Module checkbox.



  14. Click Next button to go to Step 5 and check the Module checkbox and select an application server from the server list.



  15. Click Next button to go to Step 6 and check the Web Module checkbox and select an application server from the server list.
     


  16. Click Next button to go to Step 7 that displays a summary of all the information from Step 1 through Step 6.



  17. Click on the Finish button. The console will go on to generate necessary deployment descriptors and classes. Once it finished, click on Save to Master Configuration to save the application.



  18. To start the application, click on the Applications->Enterprise Applications. Check on the check box next SimpleEjbAndViewer and then click on Start button. Now the application is started, you can go on to test it.



  19. Now you can start testing SimpleEJB and its viewer by following instructions in Testing SimpleEJB and its Viewers.

Deploy AdvancedEJB and its viewer

    Create a Database Table of mapservice

    For Oracle Database
  1. Create a MAPSERVICE table using Oracle’s Enterprise Database Manager or SQLPlus. Please note that the CONFIG column data type is CLOB.
    CREATE TABLE "user_name"."MAPSERVICE" ("NAME" VARCHAR2(255) NOT NULL,
        "TYPE" VARCHAR2(50) NOT NULL, "CONFIG" CLOB NOT NULL,
        "OUTPUT_DIR" VARCHAR2(255), "OUTPUT_URL" VARCHAR2(255),
        "IMAGE_FORMAT" VARCHAR2(5) NOT NULL, UNIQUE("NAME")) 
        TABLESPACE "table_space_name"
  2. Get JDBC driver from the Oracle server installation (usually classes12.jar file from folder like $ORACLE/ora92/jdbc/lib) and place it in the $WAS/lib/ext folder.
    For Microsoft SQL Database
  1. Create a MAPSERVICE table. Please note that the CONFIG column data type is TEXT.
    CREATE TABLE mapservice (name varchar(255), type varchar(50), config text, output_dir varchar(255), output_url varchar(255), image_format varchar(5))
  2. Download JDBC driver from Microsoft web site at http://www.microsoft.com/sql (3 jar files, msbase.jar, mssqlserver.jar and msutil.jar, extracted from either the downloaded tar file or installation program) and place them in the $WAS/lib/ext folder.

        Create JDBC provider and connection pool

  1. Click on the Resources->JDBC Providers node on the left pane of the admin console.



  2. Click on New button and
        select “Oracle JDBC Thin Driver ” from the JDBC Providers dropdown list for Oracle Database, or
        select “Microsoft JDBC driver for MSSQL Server 2000 ” from the JDBC Providers dropdown list for Microsoft SQL Server Database



  3. Click OK button to go to next screen, where you clear the Classpath entry since we put the Oracle JDBC driver or Microsoft JDBC driver for MSSQL Server 2000 in the $WAS/lib/ext folder.



  4. Click OK button to finish the creation of JDBC provider. Click on Save link to save the changes made to the configuration.



  5. Click on JDBC Providers node again and you should see the newly created the Oracle JDBC thin driver or Microsoft JDBC driver for MSSQL Server 2000.



  6. Click on Oracle JDBC Thin Driver or Microsoft JDBC driver for MSSQL Server 2000.



  7. Scroll down the right pane and click on Data Source link.



  8. Click New button to start creating a data source. Enter “Oracle Thin Data Source” or “MSSSQL 2000 Data Source” as Name, and “jdbc/OracleDS” or “jdbc/MSSQLDS” as JNDI Name.



  9. Click OK to go to next step.



  10. Click on Oracle Thin Data Source link or MSSSQL 2000 Data Source and scroll down the right pane so you can see those Additional properties of data source.

     

  11. Click on Custom Properties link to go to next screen.

     
     
  12. For Oracle database, click on URL link and enter the JDBC driver URL in form of
    jdbc:oracle:thin:username/password@machine_name:port_number:db_name

    For Microsoft SQL Server database, make sure the required properties are set correctly, e.g.,
     dabaseName: pubs
     portNumber: 1433
     serverName: localhost

     

  13. Click OK button and then save the configuration.

    14. Assembling the AdvancedEjbAdmin application

  14. If you just want to use standalone AdvancedEJB admin tool, you can skip this step. Otherwise, download the com.oreilly.servlet from http://www.servlets.com/cos/index.html and unzip the file and copy the cos.jar file to the $MOJ20/Samples/Server/J2EEServers/WebSphere/AdvanedEjbAdmin/WEB-INF/lib and then re-create the AdvancedEjbAdmin.war file that packs all the contents under $MOJ20/Samples/Server/J2EEServers/WebSphere/AdvanedEjbAdmin.

  15. Start the Application Assembly Tool and choose Application option from the popup menu. And change the Display name to “AdvancedEjbAdmin” and click Apply.

     

  16. Right click on EJB Modules to import CatalogEjb.jar file from $MOJ20/Samples/Server/J2EEServers/WebSphere folder.
     

  17. Click OK to confirm the selection of CatalogEjb.jar.



  18. Click on the EJB Modules->MOJ server EJB->Session Beans to enter JNDI Name of “ejb/Catalog” and then click Apply.


     
  19. Click on the EJB Modules->MOJ server EJB->Entity Beans->ServiceCatalogBean1 to enter JNDI Name of “ejb/ServiceCatalogBean1” in the Bindings tab and then click Apply.

     
  20. Click on the EJB Modules->MOJ server EJB->Entity Beans->ServiceCatalogBean1->Resource References to enter JNDI Name of “ejb/OracleDS” or “ejb/MSSQLDS” in the Bindings tab and then click Apply (make sure the Authentication is Application not Container in the General tab).



  21. Right click on Web Module to import the AdvancedEjbAdmin.war file from $MOJ20/Samples/Server/J2EEServers/WebSphere. And enter AdvancedEjbAdmin in the Context root: field and then click OK button

     

  22. Click on Web Modules->AdvancedEjbAdmin.war->EJB References node and click on Bindings and enter “ejb/Catalog” in the JNDI name: field.

     
  23. Save the assembled application as AdvancedEjbAdmin.ear file in the $MOJ20/Samples/Server/J2EEServers/WebSphere folder.

    24. Deploy the AdvancedEJB Admin Tool

  24. Go through steps 3 through 11 as shown in the section of  Deploy a web application as a temporary map image folder  to deploy AdvancedEjbAdmin application you just assembled (AdvancedEjbAdmin.ear).

    Assemble the AdvancedEJB and its Viewer

  25. Go through the similar steps 14 through 23 again to assemble the AdvancedEjbAndViewer application to create an AdvancedEjbAndViewer.ear file. The two files used to create the AdvancedEjbAndViewer.ear are AdvancedEjb.jar and AdvancedEjbViewer.war located at $MOJ20/Samples/Server/J2EEServers/WebSphere.
    a.    After importing both files, make sure that each bean has its JNDI name bindings set up correctly in the EJB Modules. For the session bean, set ejb/MapServiceII for MapServiceII bean and ejb/LayoutService for LayoutService bean. For the local entity bean, set ejb/ServiceCatalogBean2 for ServiceCatalogBean2 and set jdbc/OracleDS for jdbc/MapServiceDS in the Resource References node and make sure the Authentication is Application, not Container.
    b.    In the Web Modules, in the EJB References node, make sure 3 referenced beans have their JNDI bindings set as same as its value in the Name field.

    26. Deploy AdvancedEjb and its viewer

  26. Now go to WebSphere Administrative Console to deploy the AdvancedEjbAndViewer.ear file. Click on the node of Applications->Install New Application and then click on the Browse… button and navigate to the folder of $MOJ20/Samples/Server/J2EEServers/WebSphere and select AdvancedEjbAndViewer.ear file.
     


  27. Click on the Next button to go next screen in which you check the top checkbox of “Generate Default Bindings”.
     


  28. Click Next button to go to Step 1 where you check the Deploy EJBs checkbox while keeping others as default values.



  29. Click on Next button and click Next button in the Step 2 screen after selecting a Database Type which should be either MSSQLServer_2000 (or MSSQLSERVER_V7) or ORACLE_9I (or ORACLE_8).



  30. In the Step 3 screen, make sure EJB MapServiceII’s JNDI name is ejb/MapServiceII, LayoutService's JNDI name is ejb/LayoutService and ServiceCatalogBean2's JNDI name is ejb/ServiceCatalogBean2.



  31. Click Next button to go to Step 4 of Map EJB references to beans and make sure all referenced JNDI names are correct and one of the JNDI names shown in last step.



  32. Click Next button to go to Step 5 of Map resource references to resources and select a JNDI name from the dropdown list of Specify existing Resource JNDI name:, click the MOJ server EJB checkbox.



  33. Click Next button to go to Step 6 of Map virtual hosts for web modules and check the Web Module checkbox.
     


  34. Click Next button to go to Step 7 of Map modules to application servers and check the Module checkbox and select a server instance from list of Clusters and Servers.
     


  35. Click Next button to go to Step 8 of Ensure all unprotected 2.0 methods have the correct level of protection and check the EJB Module checkbox.
     


  36. Click Next button to go to Step 9 that displays a summary of all the information from Step 1 through Step 8.



  37. Click on the Finish button. The console will go on to generate necessary deployment descriptors and classes. Once it finished, click on Save to Master Configuration to save the application to master configuration file.



  38. Click on the Save button to confirm that you do want to save the application to master configuration file.



  39. To start the application, click on the Applications->Enterprise Applications. Check on the check box next to AdvancedEjbAndViewer and then click on Start button.



  40. Now you can start testing AdvancedEJB and its viewer by following instructions in Testing AdvancedEJB and its viewer with web-based admin tool, or Testing AdvancedEJB and its viewer with stand alone admin tool.

Testing JSP viewers, map EJBs and their viewers

Testing JSP Viewers

  1. Type the following URL in a browser to test JspViewer, http://localhost:<portnumber>/JspViewer/viewer.htm.

  2. Type the following URL in a browser to test JspServletViewer, http://localhost:<portnumber>/JspServletViewer/viewer.htm.

Testing SimpleEJB and its viewer

  1. Type the following URL in a browser, http://localhost:<portnumber>/SimpleEjbViewer/viewer.htm, you should see the following screen shot that display a dropdown list showing a list of map services.

     

  2. Select ‘usa1’ and click ‘Get Map Service’ and you should see the usa map shown in the browser.

     

Testing AdvancedEJB and its viewer with standalone admin tool

  1. Copy $WAS/bin/clientConfig.bat in Windows or similar script in Linix/Unix machine, to a new file say RunMOJEjbAdmin.bat for Windows or RunMOJEjbAdmin for others.

  2. Append the following line to Java's classpath in the above script:

    for Windows (after "%WAS_CLASSPATH%" )
    $MOJ20\Samples\Server\J2EEServers\WebSphere\AdvancedEjbAdminApp.jar;$WAS\installedApps\<machine_name>\AdvancedEjbAdmin.ear.ear\CatalogEjb.jar

    for Linux and Unix
    $MOJ20/Samples/Server/J2EEServers/WebSphere/AdvancedEjbAdminApp.jar:$WAS/installedApps/<machine_name>/AdvancedEjbAdmin.ear.ear/CatalogEjb.jar

  3. Run the script, you should see a Java application running with the following screen shot:
     


  4. Click on the “Add Services” on the left pane to add a new service. Fill out entries as shown in the following screen shot, where the OutputURL is only required if you are using jCIFS or NFS file system to output map images. Click on the Add button to add this new serviec to the database. If the service info has been successfully added into the database, you should see message saying that "The service 'world' has been successfully added". 



  5. Click on "List Services" on the left pane to show all deployed services.



  6. Select a service name such as world, then click on the Show Service Details button to show information about the selected service.

     
  7. To test a service, open a browser and type the URL shown in the bottom of the above screen shot (http://localhost:9080/AdvancedEjbViewer/viewer.htm?ServiceName=world).

Testing AdvancedEJB and its viewer with web-based admin tool

  1. Type the following URL in a browser to bring up the admin tool, http://localhost:9080/AdvancedEjbAdmin/index.html (note you might have different http port).

     

  2. Click on the ‘Add Service’ to add a new map service. You need to fill out entries of ServiceName, ServiceType (right now it should be ‘image’), Image Format (PNG, JPG etc), Output Directory or OutputURL (for things like jCIFS and/or NFS servers), and the service configuration ArcXML file location or the ArcXML doc. Click on Add button after you have fill out required entries. If the service is successfully added, you should see a line saying “The new service ‘world’ has been successfully added”.

     

  3. Click on the List Services to get a list of all available map services.

     

  4. Click on the service name to list the contents of the service information.
     


  5. Click on the hyperlink on the bottom of last screen "Show world Map”.
